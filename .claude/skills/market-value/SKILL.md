# 市場價值評估 + 學習迴圈

本 skill 有兩種模式,先判斷使用者意圖:

- 使用者給了**一個名詞/領域/市場**要驗證 → 進入【評估模式】
- 使用者**回報某次評估後的真實結果**、或指出評分不準要修正 → 進入【學習模式】

兩種模式都必須先完整讀取同目錄的 `MODEL.md`。

---

## 評估模式

1. 讀取 `MODEL.md`(全文)。權重順序:「六、已驗證校準」> 五維評分規則 > 通用備註。
2. 確認評估標的。若使用者只給一個大方向(如「個人品牌」),先在報告開頭用一句話定義你實際評估的範圍與同義詞群(例:個人品牌 = 個人IP / personal branding / 自媒體經營),不要反問,直接評。
3. **實跑搜尋收集訊號**。依 MODEL.md 第三章的「查詢模板」,對五個維度各跑 1–3 組 WebSearch(必要時用 WebFetch 抓關鍵頁面深看)。每個維度都要留下「我看到什麼」的證據,不能憑感覺給分。
   - 痛點強度 P、需求廣度 R、趨勢動能 T、解決成本 C、競爭門檻 K
4. **每維度打 1–5 分**,並在旁邊寫「根據哪些搜尋結果 → 為何給這分」(可追溯)。
5. **套公式算 MVS**:`MVS = (P × R × T) / (C × K)`,對照 MODEL.md 第四章的分數區間給判讀(🟢/🟡/🟠/🔴)。
6. 輸出**評估報告**,固定結構:
   - 標的定義(一句)
   - 五維評分表(維度｜分數｜證據摘要)
   - MVS 計算式與最終分數、判讀等級
   - 3 點內的「結論與下一步建議」(該做/該再深挖什麼/該放棄)
   - 一句「信心度與盲點」聲明(公開資料的限制、哪維最不確定)
7. **記錄一筆 Case** 到 `CASES.md`(編號遞增),存下這次的評分與依據,供日後校準。

## 學習模式

1. 讀取 `MODEL.md` 與 `CASES.md`。
2. 取得使用者回報的真實訊號(某標的實際做了成效如何、或某維度評分明顯高/低估、或補充了付費工具的精確數據)。
3. 做**校準分析**:
   - 當初哪一維打分偏了?偏的原因是「訊號抓錯」還是「規則本身錯」?
   - 是否需要調整某維度的評分 rubric、或調整公式權重(例:某類市場競爭門檻 K 應加重)?
   - 有沒有出現模型沒涵蓋的新訊號來源或判斷法則?
4. **更新 `CASES.md`**:在對應 Case 補上「實際結果 / 校準」段落。
5. **更新 `MODEL.md`**:
   - 調整對應維度的評分規則或查詢模板。
   - 新的判斷法則寫入「六、已驗證校準」(累積式,不覆蓋舊條目,矛盾時標註)。
   - 版本號 +0.1,並在「版本紀錄」新增一列。
6. 向使用者回報:哪裡校準了、為什麼、之後同類評估會怎麼改。

## 原則

- 所有輸出使用繁體中文、台灣用語。
- **誠實勝過好看**:公開搜尋只能做「快篩」,不是精確市調。每份報告都要標明信心度與盲點,精確月搜尋量/流量數字要靠付費工具(Ahrefs/Similarweb),沒有就別假裝有。
- 給分要保守、可追溯:寧可標「訊號不足,暫給中間值 3」也不要憑印象灌分。
- 競爭門檻 K 是雙面刃:K=1(沒人做)可能是藍海,也可能是「根本沒需求」,務必與需求廣度 R 交叉判讀,不可單看。
- 一次評估多個名詞時,各記一筆 Case、各出一份評分,最後可加一個橫向比較表。
- 更新 MODEL.md 時保持檔案結構不變(章節 + 版本紀錄),只改內容。
