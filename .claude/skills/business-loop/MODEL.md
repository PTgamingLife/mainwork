# AI 變現商業模式(活的模型）

> 這是一個「活的模型」。每次覆盤/週會都會更新此檔。派工與決策前必須完整讀取。
> 權重順序:**六(已驗證的賺錢模式)> 五(Offer 階梯)> 三(我的現況)> 一、二(通用理論)**。

**模型版本:v1.2**（2026-07-10 更新。新增 L1「週末 Vibe Code 實作包」產品大綱(借競品已驗證框架、填自己的實作);帳號自身變現數據仍待累積。）

## 一、底層邏輯(Hormozi 四支柱 — 決策時的檢查表)

### 1. 價值公式(所有 offer 的評分標準)
> **價值 =(夢想結果 × 達成機率)/(時間延遲 × 努力與犧牲)**
- **分子拉高**:客戶要的終極結果是什麼?他多相信我能幫他達成?(見證、demo、保證都在拉這裡)
- **分母壓低**:多久看到成果?過程要付出多少體力/心力?
- **我的天生優勢**:我會寫程式 + 會用 AI,能「直接幫客戶把東西做出來」→ 達成機率↑、努力犧牲↓。這是我在市場上最該放大的一格。

### 2. 無法拒絕的報價($100M Offers)
- **飢餓人群 > 好產品**:選對市場比選對產品重要。市場要有「巨大痛苦、購買力、易觸達、正在成長」。
- **價格錨定**:先出一個極高價方案,再推主打方案 → 主打像撿到便宜。
- **反轉擔憂**:用「沒成果就退費 / 保證做出 X」消除購買障礙。

### 3. 流量($100M Leads)
- **量能抵消運氣**:多數人不是方法錯,是「量」不夠。
- **內容即權威**:分享「我做過什麼」> 教「你該怎麼做」。我每天在做的 AI 實作本身就是素材。

### 4. 語錄(卡住時念一遍)
- 「你不需要更多資訊,你需要更多勇氣。」
- 「贏家與輸家的差別:當事情變無聊時,贏家還在重複基本功。」
- 「人們不是因為太貴而不買,是因為不相信你能解決問題。」
- 「你賺到的錢,與你為別人解決問題的難度成正比。」

## 二、市場檢查(飢餓人群評分,週會時重評)

目標客群:**中文圈(台灣為主)想用 AI 提效/兼職變現、但怕被課程割韭菜的上班族與家長。**

| 條件 | 現況判斷 | 分數(1–5) |
|------|---------|-----------|
| 巨大痛苦 | 想變現卻不知從何下手、怕被割韭菜、沒時間學 | 4(假設) |
| 購買力 | 上班族有可支配所得;競品 sat.cool AI 商業課定價 NT$11,000(早鳥 3,520)且在賣 → 市場願為「AI 變現」付費 | 4(競品佐證) |

| 正在成長 | AI 變現主題 2026 仍在紅利期 | 4(假設) |

> 分數皆為初始假設,靠 LOG 的留言/DM/轉換數據逐步校正。

## 三、我的現況(Positioning & Constraints)

- **帳號 / 人設**:`ailifeu`,「常常在廁所錄影片的工程師奶爸」。
- **領域**:AI 工具 / AI 工作流 / AI 變現教學。
- **獨特資產**:①真的會建 AI 工具/App(交付力強);②奶爸人設有親和力、反「高高在上導師」感;③已有兩個會自我學習的內容模型(Threads / Reels)。

- **階段**:成長中/起步(內容模型 v1.x、變現案例少)→ 迴圈重點 = **衝受眾 + 埋一個低摩擦 offer 做轉換測試**。

## 四、迴圈協定(The Loop — 這個模型如何「不斷升級」)

一個閉環,每天跑一圈、每週校一次方向:

```
派工(Dispatch)→ 執行(Execute)→ 收數據(Measure)→ 升級模型(Upgrade)→(回到派工)
   ↑———————————————— 每 7 天:週會重算 offer / 定位 ————————————————↓
```

**每日節奏模板(派工用)**:

| 時段 | 段落 | 內容 | 對應機制 |
|------|------|------|---------|
| 早 7–9 / 10–12 | 🌅 晨間派工 | 發 1 篇 Threads 或 1 支 Reels(套內容模型的高信心公式) | 流量:內容即權威 |
| 午 12–13 | 💬 午間互動 | 守留言區 + 回 3–5 則同領域熱門貼文 | 對話速度、破圈 |
| 晚 20–21 | 🧪 核心實驗 | 推進「七、實驗待辦」最上面那一個(一次只賭一變因) | 校正 offer/漏斗 |
| 晚 21 | 📊 晚間覆盤 | 回報今日數字,更新 LOG(+ 必要時 MODEL) | 自我升級 |

**升級規則(Measure→Upgrade)**:
- 假設被真實數據支持 → 信心度升級:`理論 → 觀察中 → 已驗證`。
- 連續兩次無效 → 降級或淘汰。
- 每次升級版本號 +0.1;每次週會視變動大小可 +1.0。
- 「七、實驗待辦」永遠保持非空:結案一個就補下一個。

## 五、Offer 階梯(價值階梯 — 目前假設,靠迴圈驗證)

Hormozi 邏輯:用免費內容養信任 → 低摩擦入門品成交 → 高價後端獲利。

| 層級 | 產品(假設) | 價格帶 | 價值公式重點 | 狀態 |
|------|------------|--------|-------------|------|
| L0 誘餌(免費) | Threads/Reels 內容 + 可留言換的免費資源包 | $0 | 分子:先給結果(可用的東西),建立「他真的懂」 | 進行中(內容模型已在跑) |
| **L1 入門(主推測試·v1.1 改版)** | 「一個週末,我陪你 vibe code 出一個能收款的 AI 小工具」:鎖定一個最痛的問題(TAP)→ 用 AI 做出原型 → 收到第一筆錢。主打「不是 22 堂課、不用哈佛學費、這週末就動手」 | $NT 490–1,490 | 分母:時間延遲壓到 48 小時(對手要看完 22 堂課);努力犧牲↓(vibe coding 不用會寫扣)。分子:達成機率↑(我是工程師,陪你做出來,不是丟你自己聽課)。附「週末做不出來退費」 | **理論(本季度主要驗證目標)** |
| L2 核心 | 陪跑/小班訓練營「X 週做出第一筆 AI 副業收入」(Gym Launch 模式:結果保證) | $NT 6k–15k | 分子拉滿:結果保證 + 我親自帶 | 理論(受眾/名單夠再開) |
| L3 後端 | 1:1 代建 / AI 導入顧問 | $NT 3 萬+ | 我親手交付,達成機率最高 | 理論(有需求訊號再談) |

> **當前主戰場 = L1**。迴圈的首要任務:用最小成本驗證「有沒有人願意為 L1 這種結果付低價」。
> 驗證順序(Hormozi:先確認需求再蓋產品):先測「想要訊號」(貼文問卷/DM 關鍵字)→ 再做落地頁/表單 → 再收預購/名單 → 最後才把產品做完整。

### 競品情報與對位(2026-07-10 建檔,來源:sat.cool 課程頁截圖 ×5)
- **競品**:sat.cool「搶佔 AI 紅利!哈佛講師的 22 堂商業課」,6 章 22 堂,定價 NT$11,000 / 早鳥 NT$3,520(32 折)。行銷用 Point 1–5 精緻輪播 + 試看影片 + 哈佛權威 + before/after 框架表。
- **關鍵發現(競品替我驗證了哲學)**:競品整套在教的 —— 4-1「48 小時快速驗證市場」、5-2「從賣工具到賣成果」、機會評估「TAP 痛點優先(找最痛的問題,不是最大市場)」、商業驗證「新時代 MVP:以小時為單位驗證想法」、STEP2「48 小時驗證法 → RDI 鎖定假設 → VIBE CODING → 收款 → 迭代」—— 幾乎就是本模型 Hormozi 底層的翻版。**代表這套哲學有付費市場,方向對。**
- **我的對位(反哈佛,不打權威打相反軸)**:對手贏在「哈佛權威 + 精緻 + 22 堂系統課」,售價高、交付方式是「聽」。我打不贏權威,就打相反軸 —— **反權威、工程師奶爸、真的做給你看、便宜、這週末就有成果**。對手把「VIBE CODING」放進投影片當概念講;我是工程師,能真的陪你 vibe code 出能收款的東西 → 這是我的達成機率優勢,也正中受眾「怕被割韭菜」。
- **價格錨定(Hormozi)**:L1 文案直接掛「別人 22 堂課要 3,520~11,000,我這個週末實作包 NT$XXX」,讓 L1 像撿到便宜。
- **對 L2 的意涵**:L2 陪跑訓練營可定 NT$6k–15k,貼著對手課價、但賣「我親自帶你做出成果」的更高達成機率(對手只能給影片,我給陪跑)。

### L1 產品大綱「週末 Vibe Code 實作包」(v1.2 建檔,骨架借競品框架、內容填自己的實作)

> 設計原則:把競品的「聽 22 堂」翻成「做 5 步」。每一步都有**產出物**,學員週末結束時手上有東西,不是筆記。
> 教學模板全程用競品驗證過的三步:拆真案例 → 提取可複製方法 → 套你的情境立刻做。

| 模組 | 時間 | 內容(骨架來源) | 學員產出物 | 我的 demo 素材 |
|------|------|----------------|-----------|---------------|
| M0 行前 | 週五晚 30min | 用 3 個問題選定你的題目:①誰的問題最痛(TAP 痛點優先)②你的經歷組合碰得到誰(獨有經歷=AI 無法複製的競爭力)③48h 內做不做得出最小版本(3-3 三維度判斷) | 一句話假設:「我幫 __ 解決 __,他願意付 __」(RDI 鎖定假設) | 我自己選題的真實決策過程 |
| M1 原型 | 週六上午 | Vibe coding 實戰:不靠技術團隊,用 AI 把想法變成能動的東西(對手 Point「做軟體不靠技術團隊」+ 5-3 讓 AI Agent 當執行團隊) | 一個能打開、能操作的原型(網頁/工具/bot) | 本 repo 的真實案例(line-bot、小工具、自動化腳本) |
| M2 包裝 | 週六下午 | 從賣工具到賣成果(5-2):用 Hormozi 價值公式把「我做了個工具」改寫成「我幫你拿到 __ 結果」;寫報價與保證 | 一頁 offer:成果承諾 + 價格 + 反轉擔憂保證 | 我的 L1 自己就是照這公式包的(自我示範) |
| M3 收款 | 週日上午 | 接上最簡收款與交付動線(表單/連結/轉帳都行,醜沒關係);發出第一波「想要訊號」貼文導流 | 一條可以真的收到錢的動線 + 已發出的貼文 | 我的漏斗實際長相 |
| M4 迭代 | 週日下午 | 48h 驗證循環收尾:看數據 → 判斷假設活著還是死了 → 定下一步(迭代/換題)(4-1 + STEP2 循環) | 一頁覆盤:數據、結論、下週行動 | 本模型的 LOG.md 就是活範例 |

- **交付形式(待實驗 #2/#4 校正)**:預設「自學包(模板+影片短講+檢核表)NT$490」+ 加購「週末我線上陪跑場 NT$1,490」雙檔;陪跑檔同時是 L2 訓練營的試水溫。
- **成功定義**:學員週末結束時有「能動的原型 + 一頁 offer + 能收款的動線」;最強見證 = 有學員當週末收到第一筆錢 → 直接變 L2 的招生素材。
- **不做**:22 堂式的完整知識體系、商業理論深講、精緻包裝 —— 缺的知識在做的過程中補,這正是與競品的差異。

## 七、實驗待辦(Backlog — 派工時從最上面取用,一次一個)

> 每個實驗寫成可證偽的假設 + 成功指標。結案後移到 LOG,並補新的到底部。

1. **[進行中] 需求訊號測試**:發一篇「如果有一個能讓你 7 天做出第一個 AI 變現原型的實戰包,你會想要嗎?卡在哪先告訴我」→ 看留言/DM 有多少人表態要、痛點是什麼。成功指標:≥ 15 則具體表態。
2. **[待辦] 入門品定位測試(競品對位版)**:用兩種敘事各發一篇 —— A:「7 天做出你的第一個 AI 變現原型」 vs B:「一個週末 vibe code 出能收款的東西,不用花一萬一上哈佛課」(反權威 + 價格錨定)→ 看哪種留言+分享+DM 多。成功指標:分出明顯勝方,定調 L1 主敘事與時間承諾(7 天 vs 週末/48h)。
3. **[待辦] 漏斗雛形**:做一個超簡單收名單入口(IG DM 關鍵字 or 一頁表單),把免費資源導過去。成功指標:單日 ≥ 10 個名單。
4. **[待辦] 預購/報價測試**:對名單丟 L1 報價(含價格錨定 + 退費保證),測願付。成功指標:≥ 3 筆預購或明確購買意願。
