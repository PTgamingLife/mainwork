# IG 輪播圖文設計 + 學習迴圈

本 skill 有三種模式,先判斷使用者意圖:

- 使用者給了**標題/主題 + 素材資料** → 進入【生成模式】
- 使用者上傳**類似作者的輪播作品**(或說要參考風格 / 校正基礎風格) → 進入【風格校正模式】
- 使用者上傳**自己輪播的成效截圖/數據**(或說要分析成效) → 進入【成效學習模式】

三種模式都必須先完整讀取同目錄的 `MODEL.md`。

---

## 生成模式

1. 讀取 `MODEL.md`(全文)。權重順序:「九、已驗證模式」最高 → 「八、客群輪廓」→ 「五、視覺風格系統」(基礎風格必套)→ 通用規則(一~四、六、七章)。
2. 若使用者只給標題沒給素材,直接就標題發揮,不要反問。
3. 產出**完整可上稿的輪播**,包含以下欄位:
   - **選題定位**:這則賭哪個流量機制(收藏?分享?完讀?留言?),一句話說明,並依第二章決定**張數**。
   - **封面(第 1 張)**:給 **3 個版本**,各用不同封面公式(標明 C 幾)。每個版本同時給兩層:**文案(封面大字)+ 視覺指示(版型/主色塊/配圖/字級重點)**。
     - 版本 A:模型信心度最高的公式
     - 版本 B:不同類型(如清單型 vs 痛點型)
     - 版本 C:實驗性(C9–C12 等低信心公式,幫模型收集數據)
   - **逐張腳本(第 2 張 → 最後一張)**:用表格,每列含「張次|這張的重點文案|視覺指示(版型/圖/icon/頁碼)」。遵守 MODEL.md 節奏規範(一頁一訊息、第 2 張要能當二次封面、高潮放倒數第 2 張)。
   - **最後一張(CTA 頁)**:總結金句 + 單一 CTA(依選題定位)+ 固定品牌收尾區(`@ailifeu` + slogan + 頭像)。
   - **Caption 文案**:第一行當第二鉤子,含關鍵字(SEO),把重點總結也打進去(照顧沒滑完的人),1–3 個標籤。
4. 每個封面版本後面用一行說明「這版賭的是什麼」。
5. 全程套用第五章視覺風格系統:標明主色、字體階層、版位;封面與 CTA 頁的真人出鏡,art direction 以第五章「主角人物設定」(`assets/character-reference.png`)為準,需要生圖時把角色特徵帶進 prompt 以保持同一個人;若基礎風格仍有「待校正」欄位,先用現有預設並提醒使用者可上傳範例校正。
6. 結尾附上發布操作提醒(從 MODEL.md 第七章摘 2–3 條)。
7. 檢查降權地雷清單(第六章),確保沒踩雷(尤其:封面別太多字、每張比例一致、一頁一訊息、高潮別放最後一張)。

## 風格校正模式

> 這是「先設定基礎風格,再用類似作者慢慢修正」的核心。目的是校正第五章,不是分析成效。

1. 讀取上傳的圖(Read 工具),針對每位作者/每則作品萃取**視覺風格**:
   - 版面比例、封面版型(滿版大字 / 上圖下字 / 色塊分區)。
   - 色系(主色、底色、字色,盡量給 HEX 近似值)、色彩數量。
   - 字體風格(粗黑 / 明體 / 手寫感)、標題與內文字級階層。
   - icon / 配圖風格(線性 vs 填色、真實截圖 vs 插畫)、留白程度、品牌固定元素(logo/頁碼/箭頭)位置。
   - 這位作者的封面文案用哪種公式(對應第四章 C 幾)。
2. 讀取 `MODEL.md` 與 `CASES.md`。
3. 做**風格歸納**:找出這些參考作者的共同語言、以及哪些適合套進「工程師奶爸/AI 教學」人設(乾淨科技感、可信賴,不走浮誇網紅風)。
4. **更新 `MODEL.md` 第五章**:把「基礎風格定位」的四個待校正欄位(①主色 HEX ②標題字體 ③封面版型 ④icon 風格)換成具體規格;必要時更新品牌固定元素。累積式更新,保留 v1.0 用意,矛盾時標註取捨理由。
5. **更新 `CASES.md`**:新增一筆「風格參考」型案例(標註是參考他人、非自己成效)。
6. 版本號 +0.1,「版本紀錄」新增一列。
7. 向使用者回報:這幾位作者的風格共通點、我把你的基礎風格校正成什麼(具體規格)、下一則輪播會長什麼樣。

## 成效學習模式

1. **收集這則輪播的資料**,按順序嘗試:
   - 使用者貼網址 → 先用 WebFetch 抓公開資訊(caption、帳號);IG 常擋爬蟲,抓不到就直接問使用者要:各張圖、數據(觸及/讚/留言/分享/**收藏**/完讀率)、截圖。
   - 使用者給截圖 → 用 Read 讀圖,提取:封面文案、各張重點、視覺模板、數據、留言內容與語氣。
   - 不要卡在抓不到資料;有多少分析多少,缺的欄位在 Case 裡標「未知」。**收藏數與完讀率是輪播最關鍵指標,優先跟使用者要。**
2. 讀取 `MODEL.md` 與 `CASES.md`。
3. 做**流量歸因分析**:
   - 封面是哪型?對應第四章哪條公式?封面的文案層 + 視覺層各做了什麼?
   - 這則贏在哪個機制:收藏(資訊密度/清單)?分享(共鳴)?完讀(逐步揭露)?留言(爭議/提問)?證據是收藏比、分享比還是完讀率?
   - 節奏:張數多少?第 2 張有沒有當二次封面?高潮在不在倒數第 2 張?完讀率高不高?
   - 視覺:模板一致嗎?哪一張最可能是流失點?
   - 從留言區推斷客群:誰在回、對什麼字眼有反應、誰說「已存」。
   - 有沒有模型沒涵蓋的新模式?數據不佳也要記錄並降級對應規則。
4. **更新 `CASES.md`**:依範本新增一筆「成效」型案例(Case 編號遞增),記下網址。
5. **更新 `MODEL.md`**:
   - 封面公式信心度升降級(驗證有效→升級;連兩次差→降級)。
   - 新發現的模式寫入「九、已驗證模式」。
   - 客群洞察寫入「八、客群輪廓」(累積式)。
   - 視覺上驗證有效的做法回寫第五章。
   - 版本號 +0.1,「版本紀錄」新增一列。
6. 向使用者回報:這則為什麼會(或不會)有流量(3 點內)、模型更新了什麼、下一則建議怎麼做。

## 原則

- 所有輸出使用繁體中文、台灣用語;封面與內文用「一眼看懂」的口語大字,不寫書面長句。
- 視覺指示務必具體到「照著就能排版」:寫清楚版型、主色、放什麼圖/icon、字級階層,不寫抽象形容詞。
- 一次分析多則輪播時,每則各記一筆 Case,最後統一做一次模型更新。
- 更新 MODEL.md 時保持檔案結構不變(九個章節 + 版本紀錄),只改內容。
- 風格校正模式只動第五章(與品牌元素);成效學習模式可動全模型 —— 別混淆兩者。
