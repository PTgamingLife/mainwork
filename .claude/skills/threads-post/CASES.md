## Case #1 — 2026-07-07(他人爆款,@shane412335)

**貼文摘要**:「最近看到外面在賣的 Claude 教學課程一堂要好幾千,我才發現 Anthropic 自己就出了一個完全免費的官方學習平台」— Claude 免費官方課程資源分享
**數據**:瀏覽 3.4 萬 / 讚 580 / 留言 58 / 轉發 76 / **分享 406**(截圖時間點:發文後 9 小時)
**使用的鉤子**:新型態 —— 資訊差價格對比(外面賣好幾千 vs 官方完全免費)→ 收錄為 H7
**結構**:漢堡結構變體:對比鉤子 → 資源亮點(20 堂課、附證書、很多人不知道)→ 謙遜聲明 → 行動指引(連結在留言區)→ 正向收尾

**流量歸因分析**:
1. **分享(406)是留言(58)的 7 倍** —— 資源型貼文的擴散引擎是「分享/轉發」不是留言,和模型原本「留言至上」假設不同:實用資源觸發的是「存起來+轉給朋友」行為。
2. **連結放留言區**(模型第四章規則)被完整驗證:主文零連結拿到 3.4 萬瀏覽,作者自己的連結留言還另外拿到 132 分享。
3. **「單純是興趣,不是想證明什麼」「學習 AI 我支持免費」** —— 謙遜+利他聲明消除推銷感,讓分享者無心理負擔。
4. 留言區語氣:網友簡短道謝(「謝」),作者親和回覆(「客氣了啦」)—— 作者在場感有做到。

**客群洞察**:中文圈 AI 自學者,對「免費」「官方」「證書」「別被割韭菜」極有感;付費課程貴是共同痛點。

**模型更新**:新增 H7(已驗證)、第七章新增「資源分享型公式」、修正互動權重認知(分享型貼文另計)。

## Case #2 — 2026-07-07(他人貼文,@corpenpremier,認證帳號)

**貼文摘要**:「終於有機會再用上 Claude fable5,發現 fable5 真的很強」— Claude Fable 5 做遊戲原型的使用案例教學
**數據**:瀏覽 1.5 萬(截圖時間點:發文後 14 小時;互動列未入鏡)
**使用的鉤子**:痛點喚起(「如果你曾經有一個好想法…但一直唔知點做」)+ 留言關鍵字換教學(「現在留言"神奇fable"取教學」)→ 收錄為 H8
**結構**:工具見證 → 痛點 → 具體 demo(輸入一句話 → 拆成 GDD/玩法循環/角色/關卡)→ **關鍵「小技巧」故意打碼** → 留言暗號換完整教學。掛了主題標籤「Claude fable」。

**流量歸因分析**:
1. **具體輸入輸出範例**(「我想做一隻森林冒險 RPG…」→ 拆解清單)讓能力看得見,比空講「很強」有說服力。
2. **資訊缺口設計**:把最關鍵的小技巧打碼 + 留言暗號才給 —— 製造留言動機,屬於「留言換資源」增粉手法。
3. 但瀏覽(1.5 萬 / 14hr)明顯低於 Case #1(3.4 萬 / 9hr):「留言換教學」帶推銷感、利他感低,擴散力不如純免費分享;且此手法貼近互動誘餌,有被降權風險,列為中信心度觀察中。

**客群洞察**:同為中文圈 AI 工具族(含粵語圈),吃「一步步變成原型」「不用會寫程式」的賦能敘事。

**模型更新**:新增 H8(中,觀察中)、客群輪廓補充賦能敘事偏好。

## Case #3 — 2026-07-08(自家帳號慘案,@ailifeu — 對照 Case #1 的天然實驗)

**貼文摘要**:「學 AI,英文不好真的是硬傷!(自己抄襲自己的文😂)要學 AI,Claude 官方免費課程真的很佛,上完還有證書,但全英文,外面中文課賣好幾千。所以我乾脆自己動手把課程內容整理成中文版了,重點筆記+術語對照都在留言區」— 幾乎是 Case #1 的同款公式(免費+官方+證書+資訊差+留言區)
**數據**:瀏覽 **4** / 讚 0 / 留言 8(其中多為作者自己接串的 1/4 輪播長文,非真互動)/ 轉發 0(截圖時間點:發文後 7 小時)
**使用的鉤子**:H7 變體(資訊差:外面中文課賣好幾千 vs 官方免費但全英文 → 自己整理中文版)
**結構**:漢堡結構 + 把完整課程當自我回覆貼滿留言(1/4 輪播長教學)

**流量歸因分析**:
1. **同公式、不同帳號,4 瀏覽 vs 3.4 萬瀏覽** —— 決定性變數不是內容,是**帳號**。Threads 是「帳號優先」不是「內容優先」:公式再好,也要先通過冷啟動的初始推播閘門。4 瀏覽代表這篇連追蹤者池都沒推出去,發酵機制(MODEL 一.3)在第一關就被掐死。
2. **帳號權重/主題集中度不足**:Case #1 的 @shane412335 主題集中、有既有受眾與互動history,演算法知道推給誰(興趣匹配 MODEL 一.5);@ailifeu 若是新帳號/主題分散/追蹤者少,演算法找不到對的受眾 → 初始樣本太小,對話速度模型(一.2)根本啟動不了。
3. **8 則留言是作者自己接串的整段課程長文**(把 500 頁教學貼滿留言),不是真互動;反而把話講死講滿(MODEL 四:無互動缺口),且大量自我回覆的近重複長文可能觸發「搬運/洗版」的軟性抑制訊號。
4. **開頭自削鉤子**:「(自己抄襲自己的文😂)」是內部梗,對陌生讀者是雜訊,前兩行沒鉤住 → 停留時間差(一.4)。且「英文不好+我整理中文版放留言」比 Case #1 的純利他分享多了「導流到自己東西」的味道,利他感較低。

**客群洞察**:再次驗證「免費/官方/證書/割韭菜焦慮」的痛點選得對(和 Case #1 同),證明**內容選題沒問題,問題出在帳號分發層**。

**模型更新**:第七章新增 P3(帳號權重決定天花板);第一章補冷啟動閘門認知;版本升至 v1.2。

## Case #4 — 2026-07-10(自家帳號,@ailifeu「當爸沒時間」痛點賦能文)

**貼文摘要**:「當爸之後發現,我根本沒時間好好上完一堂課,但 AI 不學一定被世界拋在後面。小孩一哭,40 分鐘教學影片分三天都看不完。所以我反過來邊做邊學,用 AI 直接生一個能動的東西出來,缺什麼再補。想把這套整理成一個 7 天實戰包」— 痛點+賦能故事,結尾二選一提問
**數據**:瀏覽 **48** / 讚 0 / 留言 0 / 轉發 0(截圖時間點:發文後 6 小時)
**使用的鉤子**:H4 真實痛點 + H6 未完成故事(當爸時間碎)+ 結尾 H5 二選一(卡點是時間還是不知從哪開始)
**結構**:痛點鋪陳 → 反轉方法(邊做邊學)→ 產品預告(7 天實戰包)→ 二選一提問收尾

**流量歸因分析**:
1. **48 瀏覽 vs 先前 4 瀏覽**,已跳出冷啟動死區 —— 但主因是**今日平台雙倍流量活動**(見 Case #5),非帳號變強(仍 0 粉)。
2. **明顯輸給同日同帳號的清單文(48 vs 168,且這篇還早發 3 小時)**:長篇痛點故事鋪陳久、資訊密度低,不利掃描擴散;結尾「7 天實戰包」帶產品導流味,利他純度低(呼應 Case #2/#3 —— 導流味壓擴散)。
3. 二選一提問設計本身沒錯,但觸及太小(48),沒累積到能引爆留言的門檻 → 0 留言。

**客群洞察**:新增「當爸/上班族時間碎片」痛點軸線 —— 沒整段時間學習是真實共鳴點,但要用「短、可掃描」的形式包裝,不能用長故事。

**模型更新**:第六章新增時間碎片客群;佐證 P3 及導流味降擴散。

## Case #5 — 2026-07-10(自家帳號,@ailifeu Claude 用途清單文,同日對照組)

**貼文摘要**:「做研究—Claude / 安排行程—Claude / 分析資料—Claude / 寫 email—Claude / 腦力激盪—Claude / 找趨勢—Claude / 生成圖片—Claude+ChatGPT / 整理文件重點—Claude+NotebookLM / 做影片—Claude+Higgsfield。我絕對不是被今天多送一倍的流量收買的!😂😂😂」(1/2)

**使用的鉤子**:清單/工具對照體(掃描式高密度)+ 幽默自嘲收尾(自曝平台在送流量)
**結構**:純列點「情境—工具」對照表 → 一句自嘲收尾。無長鋪陳。

**流量歸因分析**:
1. **同日、同 0 粉帳號、同雙倍活動,清單文 168 完勝痛點文 48(且晚發 3 小時)** —— 清單/對照體資訊密度高、可掃描、可截圖保存,天生適合分享型擴散(和 P1 資源型靠分享一致)。
2. **平台雙倍流量活動 = 冷啟動帳號的黃金窗口**:0 粉還能到 168 且破了追蹤者圈(本來 0 粉根本沒圈),證明活動能暫時抬高冷啟動天花板,瓶頸確實在分發不在內容 —— 再度驗證 P3。
3. 互動率偏低(168 瀏覽只有 2 讚 1 留言):清單體是「存起來」型內容,本來就少互動;若要衝留言需在清單裡留缺口或結尾拋問。
4. **小瑕疵**:工具名有錯字(notebbokLm→NotebookLM、higgerfield→Higgsfield、chatgpt 未大寫),AI 教學帳號的專業感會被扣分,建議修正。

**客群洞察**:再次確認客群吃「工具對照/懶人包」型內容 —— 想知道「哪個情境該用哪個 AI」,可保存性 > 故事性。

**模型更新**:第七章新增 P4(清單/工具對照體=高保存分享結構);第五章新增「留意平台流量活動日,冷啟動帳號當天集中發保存型內容」;P3 升級為已驗證(0 粉實錘);版本升至 v1.3。

## Case #6 — 2026-07-10(自家帳號,@ailifeu 情境對照懶人包，本 skill 生成的版本 A 實戰)

**貼文摘要**:「歡迎存起來,不同的事要丟給不同的 AI,用錯浪費不只一倍時間。我每天實際在用的對照,分享給跟我一開始一樣土法煉鋼的人:做研究→Perplexity、分析資料跑數字→Claude Code、寫 email→Claude、腦力激盪→ChatGPT、整理文件重點→NotebookLM、生成圖片→ChatGPT、做影片→Higgsfield。這是我目前的組合,你最常拿 AI 做哪件事?」
**數據**:瀏覽 **369** / 讚 5 / 留言 0 / 轉發 1 / **分享 2**(截圖時間點:發文後 7 小時)。帳號仍 0 粉、仍在雙倍活動日。
**使用的鉤子**:P4 清單/工具對照體 +「歡迎存起來」保存誘因開頭 + 結尾求補充提問
**結構**:保存誘因鉤子 → 情境—工具對照清單(7 項)→ 求補充提問收尾。無長鋪陳。

**流量歸因分析**:
1. **同帳號進度:4(Case #3)→ 168(Case #5)→ 369(Case #6),清單體路數完全走通**。且首次出現分享 2 + 轉發 1 —— 擴散引擎(P1/P4:靠分享轉發)實際啟動,是衝到 369 的關鍵。
2. **工具多樣化 > 全 Claude 清單**:比 Case #5 的清一色 Claude,這篇分散到 Perplexity/Claude Code/ChatGPT/NotebookLM/Higgsfield,更像「中立實用懶人包」,保存與分享價值更高,可能是擴散優於 Case #5 的原因之一。
3. **結尾求補充提問仍換不到留言(0 留言)**:再次驗證 P4「清單體天生低互動」—— 讀者是「存起來」不是「回應」。要衝留言得用更強的缺口鉤子(如版本 C「留一件不給 AI」的懸念),單純求補充力道不足。
4. **仍有小錯字**:Chatgpt 大小寫不一致、Claude Code 寫成 ClaudeCode。不影響擴散但扣專業感,持續提醒校對。

**客群洞察**:再確認「可保存的工具懶人包」是本客群最買單的型態;分享/轉發是此型態的成功指標,不是留言。

**模型更新**:P4 升級(中→高,信心提升);新增微觀洞察「工具多樣化清單 > 單一工具清單」;版本升至 v1.4。
