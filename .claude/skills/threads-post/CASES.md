## Case #1 — 2026-07-07(他人爆款,@shane412335)

**貼文摘要**:「最近看到外面在賣的 Claude 教學課程一堂要好幾千,我才發現 Anthropic 自己就出了一個完全免費的官方學習平台」— Claude 免費官方課程資源分享
**數據**:瀏覽 3.4 萬 / 讚 580 / 留言 58 / 轉發 76 / **分享 406**(截圖時間點:發文後 9 小時)
**使用的鉤子**:新型態 —— 資訊差價格對比(外面賣好幾千 vs 官方完全免費)→ 收錄為 H7
**結構**:漢堡結構變體:對比鉤子 → 資源亮點(20 堂課、附證書、很多人不知道)→ 謙遜聲明 → 行動指引(連結在留言區)→ 正向收尾

**流量歸因分析**:
1. **分享(406)是留言(58)的 7 倍** —— 資源型貼文的擴散引擎是「分享/轉發」不是留言,和模型原本「留言至上」假設不同:實用資源觸發的是「存起來+轉給朋友」行為。
2. **連結放留言區**(模型第四章規則)被完整驗證:主文零連結拿到 3.4 萬瀏覽,作者自己的連結留言還另外拿到 132 分享。
3. **「單純是興趣,不是想證明什麼」「學習 AI 我支持免費」** —— 謙遜+利他聲明消除推銷感,讓分享者無心理負擔。
4. 留言區語氣:網友簡短道謝(「謝」),作者親和回覆(「客氣了啦」)—— 作者在場感有做到。

**客群洞察**:中文圈 AI 自學者,對「免費」「官方」「證書」「別被割韭菜」極有感;付費課程貴是共同痛點。

**模型更新**:新增 H7(已驗證)、第七章新增「資源分享型公式」、修正互動權重認知(分享型貼文另計)。

## Case #2 — 2026-07-07(他人貼文,@corpenpremier,認證帳號)

**貼文摘要**:「終於有機會再用上 Claude fable5,發現 fable5 真的很強」— Claude Fable 5 做遊戲原型的使用案例教學
**數據**:瀏覽 1.5 萬(截圖時間點:發文後 14 小時;互動列未入鏡)
**使用的鉤子**:痛點喚起(「如果你曾經有一個好想法…但一直唔知點做」)+ 留言關鍵字換教學(「現在留言"神奇fable"取教學」)→ 收錄為 H8
**結構**:工具見證 → 痛點 → 具體 demo(輸入一句話 → 拆成 GDD/玩法循環/角色/關卡)→ **關鍵「小技巧」故意打碼** → 留言暗號換完整教學。掛了主題標籤「Claude fable」。

**流量歸因分析**:
1. **具體輸入輸出範例**(「我想做一隻森林冒險 RPG…」→ 拆解清單)讓能力看得見,比空講「很強」有說服力。
2. **資訊缺口設計**:把最關鍵的小技巧打碼 + 留言暗號才給 —— 製造留言動機,屬於「留言換資源」增粉手法。
3. 但瀏覽(1.5 萬 / 14hr)明顯低於 Case #1(3.4 萬 / 9hr):「留言換教學」帶推銷感、利他感低,擴散力不如純免費分享;且此手法貼近互動誘餌,有被降權風險,列為中信心度觀察中。

**客群洞察**:同為中文圈 AI 工具族(含粵語圈),吃「一步步變成原型」「不用會寫程式」的賦能敘事。

**模型更新**:新增 H8(中,觀察中)、客群輪廓補充賦能敘事偏好。

## Case #3 — 2026-07-07(他人貼文,@russ_nomad,表現偏弱的反面教材)

**貼文摘要**:「AI 的記憶到底該存 Markdown 還是 Database?🤔」— 技術二選一討論文,附螢幕翻拍照
**數據**:瀏覽 1 萬(截圖時間點:發文後 18 小時;互動列未入鏡)— 三個案例中最低(對比 Case #1 3.4萬/9hr、Case #2 1.5萬/14hr)
**使用的鉤子**:H5 立場二選一(MD 派 vs DB 派)
**結構**:問句鉤子 → 兩邊優缺列點 → **自己公布標準答案**(「後來想通了——其實不用選,正本放 MD、DB 當快取」)→ 無問句收尾 → 堆了 6 個 IG 式 hashtag

**流量歸因分析**:
1. **鉤子有效但被自己拆台**:二選一問句確實有拉力(仍拿到 1 萬瀏覽),但作者在同一篇裡給出圓滿的標準答案,把討論缺口完全關死 —— 沒人需要留言站隊了。直接驗證模型地雷「把話講死、講滿」。
2. **IG 式 hashtag 堆疊**(#人工智慧 #知識管理 #Obsidian #資料庫 #筆記軟體 #第二大腦):Threads 只吃一個主題標籤,這種搬 IG 的習慣是模型明列的降權訊號。
3. 配圖是螢幕翻拍照(拍自己問 Gemini 的畫面),資訊密度低、質感差,對停留時間幫助有限。
4. 反事實推演:同樣內容若改成「我站 MD 派,DB 派來說服我」並砍掉標準答案、只留一個標籤,留言動能會完全不同。

**客群洞察**:AI 工具族裡有一群「筆記/知識管理」重疊受眾(Obsidian、第二大腦圈),對 AI 記憶架構話題有興趣 —— 話題本身可行,是執行方式弱。

**模型更新**:H5 加註使用條件(禁止自答);地雷區補充「hashtag 堆疊」與「自問自答」實證;版本 v1.2。

## Case #4 — 2026-07-07(⭐ 自家帳號 @young_player_papa,模型生成稿實戰)

**貼文摘要**:「學 AI,英文不好真的是硬傷!」— Claude 官方免費課程中文化筆記分享,主文 + 9 則筆記自回覆做成 1/10 串文
**數據**:瀏覽 7,116 / 讚 131 / 留言 17 / 轉發 19 / **分享 132**(截圖時間點:發文後 6 小時)
**使用的鉤子**:H4 痛點抱怨開頭(「英文不好真的是硬傷」)+ H7 資訊差對比(官方免費有證書 vs 外面中文課賣好幾千)
**結構**:完整執行 P1 公式:痛點鉤子 → 資源亮點 → 對比 → 利他聲明(「純粹是自己學到順手分享」)→ 筆記放留言區 → 價值主張收尾(「學 AI 不該被英文擋在門外」)。短句斷行、口語、零 hashtag、主文零連結。

**流量歸因分析**:
1. **P1 公式在自家帳號複製成功**:分享 132 : 留言 17 ≈ 7.8:1,與 Case #1 的 7:1 幾乎同一比例 —— 「資源型貼文靠分享擴散」已跨帳號重現,升為鐵律。
2. **新招:筆記串文化(1/10)**:把中文筆記本身做成自回覆串(「1. 認識你的新同事:Claude 是誰?2/10…」),主文變成入口 hub —— 讀者往下滑筆記 = 停留時間暴增 + 每則自回覆都是可被單獨分享的內容。這是三個外部案例都沒有的原創結構,收錄為新模式 P3。
3. 開頭改用「英文不好真的是硬傷!」的痛點抱怨,比原稿 A 的敘述式開頭更有情緒衝擊 —— H4 首次實戰參與,初步有效。
4. 帳號基準線建立:7,116 瀏覽/6hr 是之後貼文的比較基準。

**客群洞察**:「英文是門檻」的共鳴在台灣 AI 自學圈成立;132 次分享代表大量讀者「存起來慢慢看」—— 教學資源的收藏價值確認。

**模型更新**:H7 → 已驗證(雙案例);H4 → 中高(首次實戰有效);P1 信心 → 高(跨帳號複製);新增 P3 筆記串文化;建立帳號基準線;版本 v1.3。
