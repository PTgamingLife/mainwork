# Threads 貼文設計模型

> 這是一個「活的模型」。每次使用者上傳貼文截圖做成效分析,就會更新此檔案。
> 生成貼文時必須完整讀取本檔,並依照權重高的規則優先套用。

**模型版本:v2.0**(2026-07-16 更新:清單體需 booster、無加持會跌回冷啟動 baseline(Case #10 僅 9 瀏覽);標籤須「熱門+對味」;新增「講人話、去英文術語」規則)

---

## 一、演算法核心機制(生成時的底層邏輯)

1. **互動權重排序:留言討論 ≈ 轉發引用 > 按讚 > 個人頁點擊 > 停留**。
   貼文的目標不是「被讚」,是「被回」。一則有 10 個來回討論的貼文,勝過 100 個讚。
2. **對話速度模型(conversation-velocity)**:發文後 30–60 分鐘的互動速度決定擴散量。30 分鐘拿 50 讚 > 24 小時拿 100 讚。
3. **發酵機制**:先推給追蹤者 → 追蹤者反應熱烈才破圈推給陌生人,發酵期 24–48 小時。
4. **停留時間**:開頭沒鉤住、被滑掉,演算法直接判定內容不吸引人。
5. **興趣匹配**:帳號主題集中在 1–2 個領域,演算法才找得到對的受眾。
6. **超過 10 個字的認真回覆**是提升權重的黃金指標;「好文」「+1」類回覆已被降權。
7. **帳號優先於內容(冷啟動閘門)**:演算法先看「這是誰發的」再決定初始推播量。同一篇內容,帳號權重/主題集中度/追蹤者互動率不同,天花板可差 1000 倍以上(Case #1 3.4萬 vs Case #3 同公式僅 4 瀏覽)。個位數瀏覽 = 沒過冷啟動閘門,問題在帳號分發層,不是內容,改文案無用。

## 二、爆款貼文結構(生成時的骨架)

**漢堡結構**(預設骨架):
- **上層|鉤子(前 1–2 行)**:反直覺觀點 / 犀利問句 / 數字衝擊。前兩行要含目標關鍵字(Threads 站內搜尋 + Google 都吃)。
- **中間|內容**:列點輸出、有節奏、口語、留白。不要把答案講完 —— 留一個缺口讓人想補充或反駁。
- **下層|收尾**:以提問結尾,邀請留言表態。

**字數與形式**:
- 全文控制在 500 字內,精華在 100–250 字之間(讓人 10 秒內看完但有內容)。
- 「像隨手打的、有人味」> 精緻官方文。太完整的教學文反而不好回應,會降低互動。
- 圖片貼文互動高 60%(9:16 直式最佳);影片 15–30 秒 + 前 3 秒鉤子。
- 標籤:一則貼文只放 1 個主題標籤(可含空格的完整片語),不硬塞。**(已驗證,Case #7)**:掛對當下熱門主題標籤(如「颱風天」)會讓貼文進入主題流,是低粉帳號破圈的重要槓桿 —— 選題若蹭到時事,務必掛對應主題標籤。**但標籤必須與內容匹配**(Case #8 反例:自介文硬掛「不想上班」,滑到的人想看厭世梗、不是自介 → 無感);硬蹭不對味的熱門標籤沒用。**標籤要「熱門 + 對味」缺一不可**(Case #10 反例:提示詞文掛冷門的「AI Threads」,對味但沒人在逛 → 無主題流量,僅 9 瀏覽);冷門/小眾 evergreen 標籤幫助有限,要挑真正有人在瀏覽的熱門主題流。
- **結尾必留互動鉤子**:任何貼文結尾都要有提問/懸念/表態邀請(Case #8 反例:自介文刪掉結尾提問 → 0 互動)。生成時不可省略收尾提問。

## 四、降權地雷(生成時絕對避開)

- ❌ 主文放外部連結(要放就放留言區)
- ❌ 互動誘餌句「同意的按讚!」「Tag 你朋友」
- ❌ 直接搬 IG 文案、全 AI 腔(太工整、太像文章)
- ❌ 過長無重點、純日記無共鳴、無觀點的純資訊
- ❌ 沒有互動缺口的貼文(把話講死、講滿)
- ❌ 未解釋的英文術語(bullet、prompt、token…)—— 與「英文不好也能學 AI」人設打架,一律改中文(bullet→重點、prompt→提示詞/指令),必要時加註解。要講人話。

## 五、發文操作建議(附在每次生成結果後提醒)

- 黃金時段:平日 7–9 AM、10 AM–12 PM、7–9 PM;週三、四、五最佳。
- 發文後 60 分鐘內守在留言區,每則留言都認真回(可提升約 42% 互動)。
- 每天 1–3 篇,重點是「保持在場」;回覆別人熱門貼文的價值 ≈ 自己發文。
- 內容輪替:痛點型 → 觀點型 → 互動型 → 教學型,不連發同型。
- **留意平台流量活動日**(如 Threads「雙倍流量」活動):冷啟動/低粉帳號當天天花板會被暫時抬高,要把握窗口集中發「保存型清單/工具對照」內容衝觸及(Case #5:0 粉當天靠活動衝到 168 瀏覽)。平時則以養帳號、互動為主。

**選題 SOP(生成貼文前先跑一次,決定要蹭哪個熱點):**

**第 0 關|人設/受眾對齊過濾器(最優先,一票否決)**:選題前先問「這篇爆了,會帶來誰?」——只追「會不會爆」是錯的,要追「爆了帶來的是不是我的目標受眾」。
- 本帳號核心領域 = **AI 學習/AI 工具教學**;目標受眾 = 想用 AI 提升自己的上班族/自學者。「爸爸/不會 code/英文普普」是**人設風味,不是主題**。
- **會爆但帶錯受眾的題目 = 傷害**(稀釋主題集中度、害演算法搞不清你是誰、吸來的粉不看你的 AI 內容)。例:育兒補助、勞權、詐騙查詢——雖有共鳴但帶來的是找補助/找法條的人,非 AI 自學者,**否決**。
- **借別人的贏家機制、但主題留在 lane 內**:賦能 demo、清單、揭穿、測驗、正義敘事都可用,但主角必須是 AI。本帳號的「正義敘事」= **反「AI 課割韭菜」、官方免費就能學**(Case #1 已驗證),不是泛用的勞權/正義。
- 過了這關的題目,才進下面的熱點來源評估。

1. **行事曆型熱點(首選,最穩)**:優先蹭「大家同時有感」的時間點,對 Threads 遠勝科技新聞。台灣年度節點與 AI 選題角度:
   - 暑假(7–8 月)→ 家長顧小孩沒時間 / 學生報告(痛點+清單)
   - 大暑酷暑(7 月底)→ 熱到不想動,AI 遠端把事做完(反差)
   - 父親節(8/8)→ 當爸的時間管理,把事交給 AI(故事+痛點)
   - 七夕(約 8 月中)→ 用 AI 生告白/文案(資源+清單)
   - 中元鬼月(8 月中下)→ 應景輕鬆梗(自嘲)
   - 開學季(8 月底–9 月初)→ 開學前設定好 AI 工作流(清單)
   - iPhone 發表(9 月初)→ 先裝這幾個免費 AI App(資源+清單)
   - 其他可套用:報稅、年終、雙 11/12 電商檔期、跨年、農曆新年
2. **臨時大事**(颱風、停電、突發新聞、政策)→ 時效鉤子威力最大但保鮮期短(數小時內要發),需即時查證熱度再抓角度。
3. **同溫層爆文**:看領域頭部帳號(如 @shane412335)最近爆的貼文 = 你受眾的真實熱點,拆解後改成自己的版本。
4. **選題來源優先序**:行事曆型(穩、可預排)> 同溫層爆文(準)> 臨時大事(爆發力強但要搶時間)。避免用歐美英文平台(HN/Mastodon/Bluesky)的科技新聞當選題 —— 與台灣中文圈受眾錯位。

**借流量戰術(低粉帳號優先做)**:與其硬發、等冷啟動,不如去領域內的爆文(尤其 H10 號召/共創大會型,如 Case #9 的 20.3 萬貼文)留言區,留一則高質量的「我用 AI 做了 X」展示,借對方流量曝光並導流個人頁轉粉。平時備 1–2 個能秀的小作品(可用 Claude Code 手搓)。

## 六、客群輪廓(由截圖分析逐步累積)

> 每次分析截圖時,從「哪種留言最多、誰在回、什麼字眼引起反應」推斷客群,寫入此區。

- 主題領域:AI 工具教學/資源(Claude 相關)— Case #1、#2
- 客群樣貌:中文圈(台灣為主、含粵語圈)AI 自學者;想用 AI 提升自己但怕被課程割韭菜;不一定會寫程式,吃「賦能敘事」(一句話變出原型/不用會 code)
- 有感的字眼/痛點:「免費」「官方」「附證書」「很多人不知道」「一堂要好幾千」(被割韭菜焦慮)、「一步步變成○○」(賦能感)、「不會 code 也能手搓出能用的東西」(賦能感極致,Case #9)
- **正義敘事的正確用法**:泛用勞權/對抗慣老闆雖台灣共鳴極強(Case #9 eagle 驗證),但**不在本帳號 lane 內**(帶來找法條的人,非 AI 自學者)。本帳號的正義敘事應鎖定 **反「AI 課割韭菜」、官方免費就能學**(Case #1 同軸),既正義又對受眾。
- 無感的內容:(待累積;初步跡象:帶推銷感的「留言換教學」「7 天實戰包」等產品導流,擴散力較弱;長篇痛點故事不利掃描)
- 新增痛點軸線:「當爸/上班族沒整段時間學習」是真實共鳴點(Case #4)—— 但要用「短、可掃描」形式包裝,別用長故事鋪陳。
- 內容偏好:吃「哪個情境用哪個 AI」的工具對照/懶人包,**可保存性 > 故事性**(Case #5)。

## 七、已驗證模式(來自實際截圖數據,權重最高)

> 此區規則來自實際截圖數據,生成時優先於第二、三章的通用規則。

**P1|資源分享型公式**(Case #1,3.4萬瀏覽/406分享,信心:高):
`資訊差對比鉤子(貴 vs 免費)→ 資源亮點列點(數字+證書+「很多人不知道」)→ 謙遜利他聲明(「單純分享,不是想證明什麼」)→ 連結放留言區 → 正向短收尾`
- 這類貼文的擴散引擎是**分享/轉發**(分享可達留言的 7 倍),成功指標看分享數,不只看留言。
- 「連結放留言區」實測有效:主文零連結照樣破圈,連結留言本身再吃一波分享。
- 利他聲明是關鍵差異:同主題貼文,無推銷感的(Case #1)瀏覽量約為帶「留言換教學」推銷感的(Case #2)2 倍以上。

**P2|具體 demo > 空講厲害**(Case #2,信心:中):
展示工具能力時,給「一句輸入 → 拆解輸出清單」的具體範例,比形容詞有說服力。可與 P1 組合使用。

**P3|帳號權重決定天花板,內容只決定天花板內的名次**(Case #1 vs #3 天然實驗 + Case #5 實錘,信心:**已驗證**):
同一套 H7 資源分享公式,@shane412335 拿 3.4 萬瀏覽,@ailifeu 只拿 4 瀏覽 —— 差別在帳號不在文案。**Case #5 實錘:@ailifeu 為 0 位粉絲帳號**,平時貼文個位數瀏覽即源於此。但同帳號在平台雙倍流量活動日衝到 168 瀏覽並破圈,證明冷啟動天花板可被「平台活動」暫時抬高,瓶頸確在分發層。可行動結論:
- **診斷順序**:先看瀏覽量級。個位數~數十瀏覽 = 帳號冷啟動問題(改文案救不了);上千瀏覽但互動差 = 才是內容/鉤子問題。
- **養帳號優先於爆文**:主題集中在 1–2 個領域(讓演算法認得你的 niche)、每天在場互動、先靠回覆別人熱門貼文累積初始受眾與權重,新帳號別期待單篇破圈。
- **不要把完整長教學貼滿自我回覆**:把話講死講滿(第四章)+ 近重複長文有洗版嫌疑,反而壓分發;留鉤子與缺口,長內容放連結或分次。
- **利他純度要夠**:「我整理中文版放留言」比 Case #1 的純免費分享多了導流味,利他感稍降;要分享就分享官方原始資源,少「導到我這」。

**P6|號召/共創型 = 最高擴散天花板**(Case #9 他人 20.3 萬實證,信心:高):
把貼文從「單向內容」變成「多人參與的舞台」——發起一個「大會/挑戰」,自己先示範,邀大家留言接力展示。這是目前看過擴散天花板最高的型態。
- **為什麼爆**:號召天然催出分享(邀朋友來炫耀)與轉發(我也參一咖);留言區高質量作品滾雪球,互動誘發互動。
- **成功要件**:①有情感/正義/賦能敘事(如「家人被慣老闆欺負,手搓勞基法小程式」)②自己先有能用的 demo 帶頭降低參與門檻 ③連結放 2/2 續文或留言區。
- **低粉帳號的用法**:0 粉自己發「大會」恐無人響應(需號召力)。反向操作——**去這類爆文的留言區,留一則高質量「我用 AI 做了 X」的展示,借流量曝光+導流個人頁**(這是 P5 轉粉機制最好的實戰場)。要有此招,平時就要備一兩個能秀的作品(可用 Claude Code 手搓小工具)。

**P5|轉粉是爆文的副產品,不靠單發自介**(Case #8 失敗驗證,信心:高):
純自我介紹型貼文 = 低擴散(79 瀏覽 vs 清單文 1,134),因為讀者拿不到能存/能轉的東西,演算法沒有擴散燃料。**轉粉的正確做法**:
- **靠爆文的留言區圈粉**:爆文當下守留言區、認真回,把互動者導向個人頁。
- **把 bio 寫好**:讓爆文帶來的人點進個人頁 3 秒內就懂「你是誰、追蹤能得到什麼」。
- **持續產出清單/資源型**:讓人「因為有用而追蹤」,不是「因為你的故事而追蹤」。
- 自介內容要放,放進 bio 或爆文的回覆串,不要單獨發一篇當主推。

**P4|清單/工具對照體 = 高保存分享結構**(Case #5→#6 連續驗證,信心:**高**):
0 粉帳號用純列點「情境—工具」清單,同帳號連續拉高:4(Case #3)→ 168(Case #5)→ 369(Case #6)→ **1,134(Case #7,時效鉤子+主題標籤加成)**,擴散引擎(分享/轉發)實際啟動。
- 清單體資訊密度高、可掃描、可截圖保存,天生適合分享型擴散(和 P1「資源型靠分享」同源)。成功指標看**分享/轉發**,不是留言。
- **⚠️ 擴散有前提:需要「初始觸及」當燃料**。清單體會放大觸及,但不會自己創造觸及。低粉帳號沒有 booster(平台活動 / 時效熱點+熱門標籤)時,會跌回冷啟動 baseline(Case #10:無加持僅 9 瀏覽;對照 Case #7 有颱風+標籤加持衝 1,134)。**1,134 是被加持出來的,不是新常態** —— 根本瓶頸仍是 0 粉(P3),養粉/借流量才是解方。
- **工具多樣化 > 單一工具清單**:分散到多個工具(Perplexity/Claude Code/ChatGPT/NotebookLM/Higgsfield)比清一色 Claude 更像「中立實用懶人包」,保存與分享價值更高(Case #6 > Case #5)。
- 適用:「哪個情境用哪個 AI」「N 個工具懶人包」這類可保存內容。開頭用「歡迎存起來」點明保存價值有效。
- **代價:互動率天生低,連「求補充」提問都換不到留言**(Case #6 仍 0 留言)。要衝留言得用更強的缺口鉤子 —— 例如「九成都能丟給 AI,只有一件我不給,你猜哪件」的懸念型,單純求補充力道不足。
- **教學帳號務必校對工具名/專有名詞**:錯字(NotebookLM、Higgsfield、Claude Code、ChatGPT 大小寫)會扣專業感與可信度。
