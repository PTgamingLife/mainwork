# 用個人 IP 幫自己加薪:從 0 到能變現的實戰課

> 講師人設:**@ailifeu — 常常在廁所錄影片的工程師奶爸**
> 領域:AI 工具實戰 / AI 變現 / 讓「英文普普、不會 code」的人也能學會用 AI
> 核心信念:**反「AI 課割韭菜」,官方免費就能學,人人都能手搓出能用的東西。**
>
> 這份課程的所有數據、案例、公式,都是我自己帳號跑出來、被真實流量驗證過的,不是網路抄來的雞湯。
> **結構說明:五個大標題,每個標題底下剛好 5 個細項,方便照著講、做投影片。**

## 第一段|開場 Hook:這個時代,沒有個人 IP 的人會怎樣?

> 一句話開場,**不要自我介紹**(這是我上課第一條規矩,也是拍片第一條——開頭自介 = 直接被滑掉)。

1. **十年前求老闆加薪,現在讓一萬個陌生人決定你值多少錢。**
   開場問句丟出去:過去你的薪水由一個主管、一間公司決定;現在你可以直接讓市場替你定價。差別只有一個——你有沒有「個人 IP」。這句話要當全場的鉤子,先勾住再說。

2. **你的專業別人看不到,就等於不存在。**
   會做同一件事的人太多了,市場不是付錢給「會做的人」,是付錢給「被看見、被信任的那個人」。沒有 IP,你永遠在跟全世界比價格;有 IP,別人是指名找你。

3. **演算法已經取代了履歷。**
   以前你的價值寫在一張 A4、由一個 HR 判斷;現在你的價值寫在內容裡、由演算法分發給對的人。不經營內容,等於把這個「免費的分發引擎」讓給別人用。

4. **AI 讓「會做事」變便宜,「被信任的人」變更貴。**
   這是我帳號一直在講的:AI 出來後,單純會操作工具的人一抓一大把,價值往下掉;真正漲價的,是那個讓大家願意聽他說「這工具怎麼用」的人。**技能會被 AI 稀釋,信任不會。IP 就是在囤信任。**

5. **把痛講到底,但一定要留出口(不要把話講死)。**
   收這一段:「最慘的不是你沒紅,是你其實很有料,世界卻不知道你存在,於是你只能拿跟沒料的人一樣的薪水。」——立刻接上承諾:「但打造 IP 的門檻從沒像今天這麼低,我一個英文普普、不會寫扣、要顧小孩、躲廁所錄影片的爸爸都做得到。」痛點必配「但有解」,製造急迫感而不是絕望。

## 第二段|IP 為什麼「一定」要有商業價值?

> 定調:「沒有商業價值的 IP,不是資產,是興趣。興趣消耗你,資產回饋你。」

1. **先打破幻覺:粉絲數 ≠ 錢。**
   我看過別人一支影片 26.4K 讚、一則貼文衝 20.3 萬瀏覽;但我自己很多貼文是個位數瀏覽——不是內容爛,是帳號還沒過冷啟動閘門。**流量會騙人,能不能換成錢才是 IP 的體檢報告。** 5000 個願意掏錢的鐵粉,勝過 10 萬個不掏錢的泛粉。

2. **商業價值是選題的過濾器,逼你講對的話給對的人。**
   當你問「這篇能不能帶來會付錢給我的人」,你就不會去追「會爆但帶錯人」的題目。我的血淚教訓:育兒補助、勞權、詐騙查詢在台灣共鳴超強、很容易爆,但帶來的是找補助、找法條的人,**不是想學 AI 的人**——這種粉越多,演算法越搞不清我是誰。會爆但帶錯受眾 = 傷害。

3. **商業價值讓你撐得久。**
   純靠愛發電的帳號,九成死在「沒回饋」的疲乏裡默默停更。有現金流(哪怕很小)的 IP,才有理由每天在場、持續產出。續航力,才是素人能不能贏的關鍵。

4. **商業價值決定你 IP 的天花板。**
   沒有變現路徑的 IP,天花板是「多一點虛榮」;有變現路徑的 IP,天花板是「多一份、甚至取代本業的收入」。一開始就想清楚要往哪變現,內容才不會白做。

5. **商業價值不等於「馬上開課賣東西」——它有四層。**
   由淺到深:①**信任資產**(有人記住你、信任你)②**導流價值**(能把流量導去某處)③**服務/接案**(有人私訊「你能不能幫我…」)④**產品/課程**(把方法打包)。我自己就反「一紅就急著賣課割韭菜」——**先給價值、把信任囤厚,商業價值是水到渠成的副產品。**

## 第三段|如何打造個人 IP:一套可複製的定位公式

> 開場:「打造 IP 不是找到你有多特別,是找到一個『你能持續講、市場又要聽』的交集。」

1. **定位公式:IP = 領域 × 人設 × 受眾的痛。**
   讓學員當場填一句話:「我是那個幫【誰】,用【什麼領域】,解決【什麼痛】的人。」三個交集鎖住,就是你的定位。用我帳號當活教材:幫「想用 AI 變現的上班族」,用「AI 工具實戰」,解決「不會 code 也想手搓出東西」的痛。

2. **領域一定要「窄」——因為演算法靠興趣推薦找人。**
   主題集中在 1–2 個領域,演算法才認得出你是誰、才找得到對的受眾。什麼都講 = 演算法搞不清你 = 誰都不推。窄不是限制,是讓對的人找得到你的前提。

3. **人設是「風味」,不是「主題」——這條分不清帳號就走鐘。**
   工程師 / 奶爸 / 英文普普 / 躲廁所錄影片,是讓人記住我、對我有共鳴的調味料;但我的主題永遠是 AI,不是育兒。人設負責「為什麼是你在講」,主題負責「你在講什麼」,兩者別混。

4. **內容支柱:定好 3–4 種內容輪流發,別每天重想。**
   ①痛點型(引共鳴、被轉發)②教學/清單型(被收藏、建專業)③觀點型(表態、拉留言)④展示型(證明你真的會)。四型輪替、不連發同型。**鐵律:每一篇都要能一句話回答「觀眾看完拿到什麼」**——我發過日記型心得文,24 讚 0 留言,因為觀眾拿不到東西。

5. **打造 IP 的三階段,給自己一個時間感。**
   ①**養帳號期**:主題集中、每天在場、靠留言別人熱門貼文累積初始受眾,別期待單篇爆。②**累積信任期**:穩定產出有用內容,讓人「因為有用而追蹤」,備 1–2 個能秀的小作品(我都用 Claude Code 手搓)。③**變現期**:有人開始私訊你、指名你,商業價值自然浮現。**反雞湯:同一套公式,頭部帳號拿 3.4 萬瀏覽、我 0 粉帳號只拿 4 瀏覽——差別在帳號不在文案,地基騙不了人。**

## 第四段|IG 如何創造流量:底層邏輯與平台規則

> 開場:「先講最反直覺的一句:IG 的目標不是讓你紅,是讓用戶一直滑下去。你要做的,是變成那個讓人停不下來的內容。」

1. **權重排序:完播/觀看時間 ≈ 35% > 收藏 25% > DM 轉發 20% > 讚 只剩 5%。**
   這是全場最重要的一張黑板。翻成人話:**你的影片目標不是「被讚」,是「被看完、被存、被傳給某個人」。** 一支 100 讚沒人看完的,輸給一支 20 讚但大家看完還轉發的。讚已經幾乎沒用了。

2. **五條刻進骨子的機制。**
   ①**第 3 秒是生死線**:滑過 3 秒還在看 → 演算法判定鉤子有效、往外推。②**重播 = 超高價值訊號**:短片(7–15 秒)配「結尾接回開頭」的循環設計最容易騙到重播。③**DM 轉發權重是讚的 3–5 倍**:設計「傳給你那個○○的朋友」。④**發布後 30–60 分鐘互動速度**決定推不推給非粉絲(進 Explore)。⑤**興趣推薦取代粉絲導向**:主題集中才被找到,Reels 觸及約照片 2.5 倍。

3. **鉤子三層同時進攻(85% 的人關聲音在看)。**
   ①**口白層**:第一句就是鉤子且含關鍵字(IG 搜尋 + Google 都吃)。②**畫面層**:第一幀就要有動作/結果/強烈畫面,**先秀成果再講過程,靜止開頭是死刑**。③**字卡層**:3–7 個大字壓縮承諾、高對比、放安全區。好用公式抄走:段位對照(「用 AI 的 6 個段位,你在哪一級」一個框架同吃完播/收藏/留言)、負面警告、反直覺斷言、秘密內幕。

4. **降權地雷:一做就被扣分(每條我都用血換過)。**
   ❌開頭自我介紹鋪陳 ❌帶浮水印的轉載(TikTok logo 直接降權)❌非 9:16 直式/畫質差 ❌「按讚訂閱分享」連環 CTA(一支只下一個)❌硬塞 30 個 hashtag(1–3 個精準就好)❌音樂蓋過口白、沒字幕 ❌把答案在 hook 就講完。賺流量之前,先別被扣分。

5. **我的三個真實流量教訓 + 冷啟動診斷公式(這段最有說服力)。**
   ①**6–7 秒魔咒**:兩支題材不同的影片平均觀看都卡 6–7 秒,元凶是全程單鏡、零視覺變化 → 解法:**主體每 2–3 秒一個視覺變化**,靜止超過 4 秒就危險。②**猛料前置 0–3 秒**:我把免費資源埋到第 23 秒 → 4 天 8 讚躺平;最捨不得先講的那張牌,直接放第一句。③**留言關鍵字機制**:給交換條件(「留言○○,私訊完整教學給你」)可把留言率從 1–3% 拉到 **35%**。**診斷公式**:觀看低+互動低 = 帳號冷啟動(養帳號,改文案救不了);觀看高+互動低 = 題材對包裝錯(修 hook/節奏);觀看低+互動高 = 包裝對選題錯(修選題/標籤)。

## 第五段|七天作業:接下來 7 天,打造你 IP 的第一塊地基

> 交代:「聽完不做等於沒上。這七天不是叫你爆紅,是把地基打下去。每天 30–60 分鐘,做完你就有一個能跑的 IP 雛形。」(七天各一個交付物,底下這 5 條是每天都要遵守的共同守則。)

1. **Day 1–2|先定位、再挖痛點。**
   Day 1:寫出「我是那個幫【誰】,用【領域】,解決【什麼痛】的人」+ 選定唯一領域(1–2 個)+ 三個人設關鍵字。Day 2:寫出目標受眾「一天中跟你領域有關的 3 個具體煩惱」及他們現在怎麼笨笨地解決——這就是你的選題礦。

2. **Day 3–4|排內容支柱、整帳號基建。**
   Day 3:定 3–4 個內容支柱,每個支柱先想 2 個題目,湊出 8 個選題進題庫。Day 4:把 bio 寫到「陌生人 3 秒內就懂你是誰、追蹤能得到什麼」,頭像清楚、名稱含關鍵字,並準備 1 個「能秀的小作品」當展示素材。

3. **Day 5|寫出第一支腳本,套三層 Hook。**
   從題庫挑 1 題,寫成 15–30 秒腳本,必含:①口白層鉤子(第一句含關鍵字)②畫面層(第一幀秀成果)③字卡層(3–7 字)。自我檢查:主體有沒有「每 2–3 秒一個視覺變化」?結尾是不是「只有一個 CTA」?

4. **Day 6|發佈 + 守留言區,實測冷啟動。**
   把腳本拍出來發佈(9:16、上字幕、1 個 CTA、1–3 個精準標籤)。**發佈後 60 分鐘守在留言區**衝互動速度;同一天去領域內一則熱門貼文的留言區,留一則高質量、有料的留言(借流量、被看見)。

5. **Day 7|複盤,用診斷公式讀對的指標(不看讚)。**
   打開後台記下:觀看、平均觀看時間、略過率、收藏、分享。用第四段診斷公式判斷你現在是「帳號冷啟動」還是「內容包裝」問題,然後寫下下一支**只改的那一件事**(別全改)。收尾呼應開場:「演算法不會等你準備好——我一個躲廁所錄影片的爸爸都做得到,接下來這七天,換你了。」
